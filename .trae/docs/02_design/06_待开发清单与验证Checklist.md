## 3. 需求落地验证 Checklist

### ✅ 代码修改验证

```
- [ ] 代码实现是否严格遵循需求文档描述？
- [ ] 是否有对应的单元测试用例覆盖核心逻辑？
- [ ] 代码审查是否通过（至少 1 人 review）？
- [ ] 是否存在未使用的变量/函数（死代码）？
- [ ] 代码格式是否符合项目规范（ESLint 检查通过）？
```

### ✅ 接口验证

```
- [ ] API 响应格式是否符合技术规范定义？
- [ ] HTTP 状态码是否正确（200/201/400/401/403/404）？
- [ ] 边界条件是否正确处理（空数据、异常输入、超大值）？
- [ ] 权限控制是否正确生效（不同角色返回不同数据范围）？
- [ ] 错误响应是否包含友好的错误信息？
```

### ✅ 前端验证

```
- [ ] UI 展示是否与需求描述/设计稿一致？
- [ ] 交互逻辑是否符合设计规范（弹窗、Toast、加载状态）？
- [ ] 响应式布局是否正常（桌面/平板/移动端）？
- [ ] 输入验证是否完整（必填、格式、长度校验）？
- [ ] 无障碍访问是否考虑（键盘导航、屏幕阅读器）？
```

### ✅ 业务验证

```
- [ ] 核心业务场景是否能完整走通（端到端测试）？
- [ ] 数据计算结果是否正确（与预期值对比）？
- [ ] 与其他模块的联动是否正常（无回归问题）？
- [ ] 性能是否符合要求（页面加载 < 2s，接口响应 < 500ms）？
- [ ] 安全性是否达标（无 SQL 注入、XSS 漏洞）？
```

### ✅ 文档同步验证

```
- [ ] 需求文档是否已更新到最新版本？
- [ ] API 文档（技术规范）是否同步更新？
- [ ] 变更履历是否记录完整？
- [ ] 本待开发清单是否已更新状态？
- [ ] 测试用例文档是否同步更新？
```
