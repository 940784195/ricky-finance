## 3. 业务需求详细定义

### 3.1 认证模块 (REQ-AUTH)

#### REQ-AUTH-01: 用户登录认证
- **需求描述**: 用户使用用户名和密码登录系统，验证通过后返回 JWT Token
- **优先级**: P0
- **关联需求**: 无
- **验收标准**:
  - 有效凭证登录成功，返回 Token
  - 无效用户名返回 401
  - 无效密码返回 401
  - 缺少必填项返回 400

#### REQ-AUTH-02: 登录状态保持
- **需求描述**: 登录后 Token 在有效期内保持登录状态
- **优先级**: P1
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - Token 在 24h 内有效
  - 过期后需要重新登录

#### REQ-AUTH-03: 安全退出功能
- **需求描述**: 用户可主动退出登录，清除本地 Token
- **优先级**: P1
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 退出后清除本地存储
  - 退出后访问受保护资源返回 401

### 3.2 权限模块 (REQ-ROLE)

#### REQ-ROLE-01: 系统管理员权限
- **需求描述**: 管理员可查看所有数据，管理所有用户资产记录
- **优先级**: P0
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 可查看所有家庭记录
  - 可增删改查所有用户资产
  - 不可直接修改家庭数据（仅查看）

#### REQ-ROLE-02: 户主权限
- **需求描述**: 户主可管理本家庭所有成员和资产
- **优先级**: P0
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 可管理家庭成员
  - 可查看/管理本家庭所有资产
  - 可添加自定义资产类型

#### REQ-ROLE-03: 家庭成员权限
- **需求描述**: 成员仅可管理自己的资产
- **优先级**: P0
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 仅查看自己的资产
  - 仅可添加/编辑自己的记录
  - 不可查看其他成员记录

### 3.3 家庭管理模块 (REQ-FAM)

#### REQ-FAM-01: 家庭信息管理
- **需求描述**: 户主可修改家庭名称，查看家庭基本信息
- **优先级**: P0
- **关联需求**: REQ-ROLE-02
- **验收标准**:
  - 户主可修改本家庭名称
  - 修改后家庭名称正确更新
  - 成员不可修改家庭信息

#### REQ-FAM-02: 系统管理员查看所有家庭
- **需求描述**: 管理员可查看系统中所有家庭列表及详情
- **优先级**: P0
- **关联需求**: REQ-ROLE-01
- **验收标准**:
  - 可查看所有家庭列表
  - 可查看指定家庭详情
  - 家庭不存在返回 404

### 3.4 成员管理模块 (REQ-MEM)

#### REQ-MEM-01: 家庭成员CRUD
- **需求描述**: 户主可对家庭成员进行增删改查操作
- **优先级**: P0
- **关联需求**: REQ-FAM-01
- **验收标准**:
  - 户主可添加新成员
  - 户主可编辑成员信息
  - 户主可删除成员（级联删除资产记录）
  - 户主可查看成员列表及详情
  - 成员姓名必填

#### REQ-MEM-02: 成员权限控制
- **需求描述**: 不同角色对成员数据的访问控制
- **优先级**: P0
- **关联需求**: REQ-MEM-01
- **验收标准**:
  - admin可查看所有成员
  - head可查看本家庭成员
  - member仅可查看自己
  - 越权访问返回 403

### 3.5 资产记录模块 (REQ-REC)

#### REQ-REC-01: 新增资产记录
- **需求描述**: 用户可录入单条资产记录
- **优先级**: P0
- **关联需求**: REQ-ROLE-02, REQ-ROLE-03
- **验收标准**:
  - 必填项完整时可成功创建
  - 缺少必填项返回 400
  - 管理员必须指定成员
  - 户主可选择成员（默认自己）

#### REQ-REC-02: 编辑资产记录
- **需求描述**: 用户可修改已有记录
- **优先级**: P0
- **关联需求**: REQ-REC-01
- **验收标准**:
  - 可更新部分字段
  - 不可修改不属于自己的记录（403）
  - 不存在的记录返回 404

#### REQ-REC-03: 删除资产记录
- **需求描述**: 用户可删除单条记录
- **优先级**: P0
- **关联需求**: REQ-REC-01
- **验收标准**:
  - 可删除自己的记录
  - 不可删除不属于自己的记录（403）
  - 不存在的记录返回 404

#### REQ-REC-04: 查看记录详情
- **需求描述**: 用户可查看单条记录完整信息
- **优先级**: P0
- **关联需求**: REQ-REC-01
- **验收标准**:
  - 可查看自己的记录详情
  - 户主可查看家庭所有记录
  - 不存在的记录返回 404

#### REQ-REC-05: 资产名称自动补全
- **需求描述**: 输入资产名称时自动提示历史名称
- **优先级**: P1
- **关联需求**: REQ-REC-01
- **验收标准**:
  - 聚焦时显示历史名称列表
  - 输入时实时过滤
  - 支持关键词搜索

#### REQ-REC-06: 记录筛选与搜索
- **需求描述**: 支持多条件筛选和关键词搜索
- **优先级**: P1
- **关联需求**: REQ-REC-04
- **验收标准**:
  - 可按成员筛选
  - 可按类型筛选
  - 可按名称筛选
  - 可按日期范围筛选
  - 支持关键词搜索

#### REQ-REC-07: 同一资产名称记录更新规则
- **需求描述**: 同一资产名称的最新记录覆盖历史记录
- **优先级**: P0
- **关联需求**: REQ-REC-01, REQ-STAT-01
- **验收标准**:
  - 统计时取最新记录
  - 不重复累加同一资产
  - 历史记录保留可查

#### REQ-REC-08: 记录状态管理
- **需求描述**: 资产记录支持valid/pending/invalid三种状态
- **优先级**: P0
- **关联需求**: REQ-REC-01
- **验收标准**:
  - 新增记录默认状态为valid
  - 统计时仅计算valid记录
  - 支持按状态筛选

### 3.6 统计模块 (REQ-STAT)

#### REQ-STAT-01: 家庭总资产统计
- **需求描述**: 按资产名称去重，统计最新记录总和
- **优先级**: P0
- **关联需求**: REQ-REC-07
- **验收标准**:
  - 同一资产仅统计最新记录
  - 不同资产分别取最新后累加
  - 仅统计有效记录

#### REQ-STAT-02: 资产类型分布
- **需求描述**: 按类型分组统计资产分布
- **优先级**: P1
- **关联需求**: REQ-STAT-01
- **验收标准**:
  - 按类型正确分组
  - 每组内按名称去重
  - 显示金额和占比

#### REQ-STAT-03: 资产趋势分析
- **需求描述**: 展示资产价值随时间变化趋势
- **优先级**: P2
- **关联需求**: REQ-STAT-01
- **验收标准**:
  - 支持多时间范围
  - 显示增长/下降趋势
  - 鼠标悬停显示详情

#### REQ-STAT-04: 统计指标完整性
- **需求描述**: 仪表盘应展示完整的统计指标
- **优先级**: P1
- **关联需求**: REQ-STAT-01
- **验收标准**:
  - 返回totalValue, totalRecords, memberCount
  - 返回activeMembers, monthlyNew, pendingCount
  - 返回totalChange, totalRate

### 3.7 资产类型模块 (REQ-TYPE)

#### REQ-TYPE-01: 资产类型管理
- **需求描述**: 户主可自定义资产类型
- **优先级**: P1
- **关联需求**: REQ-ROLE-02
- **验收标准**:
  - 可添加自定义类型
  - 可编辑自定义类型
  - 可删除未使用的类型
  - 系统预设类型不可删除

#### REQ-TYPE-02: 恢复默认类型
- **需求描述**: 户主可恢复系统预设的默认资产类型
- **优先级**: P1
- **关联需求**: REQ-TYPE-01
- **验收标准**:
  - 可恢复缺失的默认类型
  - 不覆盖已存在的默认类型
  - 使用事务保证原子性

### 3.8 导出模块 (REQ-EXP)

#### REQ-EXP-01: CSV导出功能
- **需求描述**: 支持将资产记录导出为CSV格式
- **优先级**: P1
- **关联需求**: REQ-REC-04
- **验收标准**:
  - 支持按筛选条件导出
  - 包含UTF-8 BOM
  - 权限控制正确

#### REQ-EXP-02: Excel导出功能
- **需求描述**: 支持将资产记录导出为Excel格式
- **优先级**: P1
- **关联需求**: REQ-EXP-01
- **验收标准**:
  - 生成有效的.xlsx文件
  - 包含正确的数据格式
  - 权限控制正确

### 3.9 系统设置模块 (REQ-SET)

#### REQ-SET-01: 修改密码
- **需求描述**: 用户可修改自己的登录密码
- **优先级**: P0
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 需提供当前密码验证
  - 新密码需确认
  - 修改成功后需重新登录

#### REQ-SET-02: 家庭设置
- **需求描述**: 户主可修改家庭名称等设置
- **优先级**: P1
- **关联需求**: REQ-FAM-01
- **验收标准**:
  - 可修改家庭名称
  - 修改后即时生效

### 3.10 安全模块 (REQ-SEC)

#### REQ-SEC-01: 密码安全策略
- **需求描述**: 密码使用bcrypt加密存储
- **优先级**: P1
- **关联需求**: REQ-AUTH-01
- **验收标准**:
  - 密码不以明文存储
  - 使用bcrypt hash格式
  - salt rounds = 10

#### REQ-SEC-02: SQL注入防护
- **需求描述**: 全部使用参数化查询防止SQL注入
- **优先级**: P0
- **关联需求**: 所有数据库操作
- **验收标准**:
  - 所有查询使用?占位符
  - 动态WHERE子句安全构建

#### REQ-SEC-03: 数据隔离
- **需求描述**: 三层数据隔离策略
- **优先级**: P0
- **关联需求**: REQ-ROLE-01~03
- **验收标准**:
  - admin可见全局数据
  - head仅可见本家庭数据
  - member仅可见个人数据

### 3.11 性能模块 (REQ-PERF)

#### REQ-PERF-01: API响应性能
- **需求描述**: API响应时间满足性能指标
- **优先级**: P2
- **关联需求**: 所有API
- **验收标准**:
  - 简单查询 < 100ms
  - 统计聚合 < 500ms
  - 数据库写入 < 50ms

## 4. 测试数据设计

### 4.1 认证模块测试数据 (TD-AUTH)

#### TD-AUTH-01: 有效登录凭证
```json
{
  "username": "head",
  "password": "head123"
}
```
- **数据类型**: 正常场景
- **预期结果**: HTTP 200, 返回 JWT Token

#### TD-AUTH-02: 无效用户名
```json
{
  "username": "nonexistent",
  "password": "head123"
}
```
- **数据类型**: 异常场景
- **预期结果**: HTTP 401, 错误提示

#### TD-AUTH-03: 无效密码
```json
{
  "username": "head",
  "password": "wrongpassword"
}
```
- **数据类型**: 异常场景
- **预期结果**: HTTP 401, 错误提示

#### TD-AUTH-04: 缺少用户名
```json
{
  "password": "head123"
}
```
- **数据类型**: 边界条件
- **预期结果**: HTTP 400, 缺少必填项提示

#### TD-AUTH-05: 缺少密码
```json
{
  "username": "head"
}
```
- **数据类型**: 边界条件
- **预期结果**: HTTP 400, 缺少必填项提示

#### TD-AUTH-06: Token 状态验证
```
Token: valid_jwt_token (24h有效期)
```
- **数据类型**: 正常场景
- **预期结果**: 有效期内可访问受保护资源

#### TD-AUTH-07: 退出登录
```
操作: 调用 logout API 或清除本地存储
```
- **数据类型**: 正常场景
- **预期结果**: 清除 Token，跳转登录页

### 4.2 权限模块测试数据 (TD-ROLE)

#### TD-ROLE-01: 管理员查看所有记录
```
角色: admin
请求: GET /api/records
```
- **数据类型**: 正常场景
- **预期结果**: 返回所有家庭的所有记录

#### TD-ROLE-02: 管理员管理所有记录
```
角色: admin
操作: POST/PUT/DELETE /api/records
```
- **数据类型**: 正常场景
- **预期结果**: 可增删改查所有记录

#### TD-ROLE-03: 管理员添加记录必须指定成员
```json
{
  "role": "admin",
  "data": {
    "type": "stock",
    "name": "测试股票",
    "value": 100000,
    "date": "2026-06-01"
  }
}
```
- **数据类型**: 边界条件
- **预期结果**: HTTP 400, 必须指定 memberId

#### TD-ROLE-04: 户主查看家庭记录
```
角色: head
请求: GET /api/records
```
- **数据类型**: 正常场景
- **预期结果**: 返回本家庭所有成员记录

#### TD-ROLE-05: 户主为成员添加记录
```json
{
  "role": "head",
  "data": {
    "memberId": 2,
    "type": "stock",
    "name": "测试股票",
    "value": 100000,
    "date": "2026-06-01"
  }
}
```
