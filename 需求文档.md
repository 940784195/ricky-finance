# Ricky Finance - 资产管家 需求文档

## 1. 项目概述

### 1.1 项目名称
资产管家 (Ricky Finance)

### 1.2 项目定位与目标
**软件定位**: 面向家庭用户的资产管理系统

**项目目标**: 构建一个家庭资产管理系统,用于记录、管理、分析家庭成员的各类资产,提供数据可视化、报表导出等功能。支持户主管理家庭成员,管理员查看所有家庭数据。

### 1.3 技术栈
- 后端: Node.js + Express
- 数据库: SQLite (better-sqlite3)
- 前端: HTML + CSS + JavaScript
- 图表库: Chart.js
- 测试: Jest + Playwright
- AI助手: TRAE Agents/Skills/Tools

## 3. 详细功能需求

### 3.1 用户认证与权限管理

#### 3.1.1 用户角色与登录
- 支持三种用户角色:
  - 系统管理员
  - 户主
  - 家庭成员
- 默认测试账号:
  - 系统管理员: admin / admin123
  - 户主: head / head123
  - 家庭成员: member / member123
- 登录状态保持
- 安全退出功能

#### 3.1.2 权限控制
- **系统管理员**:
  - 可查看所有户主、家庭成员的信息
  - 可查看所有家庭的资产数据
  - 可管理系统配置
  - 不可直接修改家庭数据(仅查看)
  
- **户主**:
  - 可管理(增加、编辑、删除)家庭成员
  - 可查看和管理本家庭所有成员的资产数据
  - 可添加、编辑、删除资产记录
  - 可查看家庭资产分析报告
  
- **家庭成员**:
  - 仅查看自己的资产数据
  - 可添加自己的资产记录
  - 可编辑自己的资产记录

### 3.2 家庭与成员管理

#### 3.2.1 家庭信息字段
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | 数字 | 是 | 家庭唯一标识 |
| familyName | 字符串 | 是 | 家庭名称 |
| headId | 数字 | 是 | 户主ID |
| createdAt | 日期时间 | 是 | 创建时间 |

#### 3.2.2 家庭成员信息字段
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | 数字 | 是 | 成员唯一标识 |
| familyId | 数字 | 是 | 所属家庭ID |
| name | 字符串 | 是 | 成员姓名 |
| shortName | 字符串 | 否 | 成员简称(用于头像显示) |
| role | 字符串 | 是 | 角色(head/member) |
| createdAt | 日期时间 | 是 | 创建时间 |

#### 3.2.3 家庭管理功能(户主权限)
- **新增家庭成员**: 录入新家庭成员的基本信息
- **编辑家庭成员**: 修改家庭成员信息
- **删除家庭成员**: 删除家庭成员(级联处理关联的资产记录)
- **查看家庭成员详情**: 查看成员详细信息及资产记录
- **家庭成员列表展示**: 以卡片形式展示所有家庭成员
- **家庭统计**: 家庭成员数、家庭总资产等

#### 3.2.4 家庭管理功能(系统管理员权限)
- **查看所有家庭列表**: 查看系统中所有家庭信息
- **查看家庭详情**: 查看指定家庭的完整信息和资产数据
- **家庭统计**: 家庭总数、系统总资产等

---

### 3.3 资产记录管理

#### 3.3.1 资产记录字段
| 字段名 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| id | 数字 | 是 | 记录唯一标识 |
| memberId | 数字 | 是 | 关联家庭成员ID |
| familyId | 数字 | 是 | 所属家庭ID |
| type | 字符串 | 是 | 资产类型 |
| name | 字符串 | 是 | 资产名称 |
| value | 数字 | 是 | 资产价值(元) |
| previousValue | 数字 | 否 | 上期价值(用于计算变化率) |
| date | 日期 | 是 | 记录日期 |
| status | 字符串 | 是 | 状态(valid/invalid/pending) |
| note | 文本 | 否 | 备注说明 |
| createdAt | 日期时间 | 是 | 创建时间 |

#### 3.3.2 资产类型管理
**资产类型定义**:
- 资产类型为可自定义项,系统提供预设选项作为引导
- 户主可自定义添加、编辑、删除资产类型
- 每个资产类型包含:类型标识、显示名称、颜色

**预设资产类型(引导选项)**:
| 类型值 | 显示名称 | 颜色 |
|--------|----------|------|
| stock | 股票 | #f59e0b |
| fund | 基金 | #3b82f6 |
| bond | 债券 | #10b981 |
| realestate | 房地产 | #ec4899 |
| cash | 现金 | #6366f1 |
| other | 其他 | #6b7280 |

**资产类型管理功能(户主权限)**:
- 添加自定义资产类型
- 编辑已有资产类型(名称、颜色)
- 删除自定义资产类型(检查是否有记录使用)
- 恢复默认预设类型

#### 3.3.3 资产记录功能
- **新增记录**: 手动录入单条资产记录
- **编辑记录**: 修改已有记录
- **删除记录**: 删除单条记录
- **查看记录详情**: 查看记录的完整信息
- **批量导入**: 支持Excel/CSV文件批量导入资产记录
- **批量导出**: 支持Excel/CSV格式导出数据

#### 3.3.4 记录筛选与搜索
- 按家庭成员筛选
- 按资产类型筛选(包括自定义类型)
- 按状态筛选(有效/无效/待审核)
- 按日期范围筛选
- 关键词搜索(成员名、资产名称)

#### 3.3.5 变更时间线
- 以时间轴形式展示资产变更历史
- 突出显示增长/下降趋势
- 支持查看每次变更的详细信息

### 3.4 资产概览与数据分析

#### 3.4.1 关键指标统计
- **户主视角**:
  - 家庭总资产价值及变化率
  - 家庭成员数及活跃成员数
  - 家庭资产记录总数
  - 本月新增记录数
  - 待处理记录数
  
- **系统管理员视角**:
  - 系统总资产价值
  - 家庭总数
  - 成员总数
  - 系统资产记录总数

#### 3.4.2 数据可视化
- **资产趋势图**: 
  - 折线图展示资产总值随时间的变化
  - 支持查看近6个月、近1年、全部时间的数据
  - 展示总体增长/下降趋势
  
- **资产配置图**:
  - 饼图展示各类资产的占比
  - 直观显示资产分布情况

#### 3.4.3 可定制化仪表盘
- 支持添加多种类型的图表
- 图表位置可拖拽调整
- 图表尺寸可自定义
- 支持删除不需要的图表

## 4. AI辅助功能需求

### 4.1 数据分析代理
- **功能描述**: 分析客户资产变化趋势,生成智能洞察
- **能力**:
  - 趋势分析:识别资产增长/下降趋势
  - 对比分析:跨客户、跨时间的资产对比
  - 异常识别:检测资产数据的异常波动
  - 预测分析:基于历史数据预测未来趋势
  - 洞察生成:自动生成业务洞察报告

### 4.2 异常检测代理
- **功能描述**: 监控资产数据,识别异常变化并发出警报
- **能力**:
  - 阈值监控:检测超过预设阈值的变化
  - 模式识别:识别异常的数据模式
  - 趋势异常:检测与历史趋势不符的变化
  - 预警机制:自动触发异常预警

### 4.3 数据同步代理
- **功能描述**: 确保桌面端与移动端数据一致性
- **能力**:
  - 实时同步:检测变更并立即同步
  - 冲突解决:处理数据同步冲突
  - 增量同步:仅同步变化的数据
  - 离线支持:支持离线操作后的同步

### 4.4 提醒通知代理
- **功能描述**: 提醒客户/管理员定期更新资产数据
- **能力**:
  - 定期提醒:按计划发送数据更新提醒
  - 逾期提醒:提醒未及时更新的用户
  - 事件通知:资产变化达到阈值时的通知
  - 多渠道推送:支持邮件、短信、APP推送

### 4.5 报告生成代理
- **功能描述**: 自动生成格式化的资产报告
- **能力**:
  - 定期报告:按周/月/季度生成报告
  - 自定义报告:根据需求生成特定报告
  - 多格式输出:支持PDF、Excel、HTML
  - 模板支持:支持多种报告模板

## 6. 数据存储需求

### 6.1 数据库设计
- 使用SQLite作为主要数据库
- 数据表:
  - users: 用户账号表
  - families: 家庭信息表
  - members: 家庭成员表
  - asset_types: 资产类型表(支持自定义)
  - records: 资产记录表

### 6.2 数据文件
- server/data/families.json: 家庭数据(备份)
- server/data/members.json: 家庭成员数据(备份)
- server/data/asset_types.json: 资产类型数据(备份)
- server/data/records.json: 资产记录(备份)
- server/data/users.db: SQLite数据库文件
